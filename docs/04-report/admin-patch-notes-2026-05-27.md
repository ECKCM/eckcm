## 1. ✅ 체크인 시스템 전면 개편

체크인 화면이 크게 바뀌었습니다. (**사이드바 → Check-in**)

**새로 생긴 것**
- **스캔 세션(Scan Session)**: 체크인을 시작하기 전에 "세션"을 **시작(Start)** 하고, 끝나면 **종료(Stop)** 합니다. 세션별로 누가·언제·몇 명을 체크인했는지 기록이 남습니다.
- **실시간 현황 대시보드**: 체크인이 들어오는 즉시 화면에 반영됩니다. 여러 명이 동시에 스캔해도 함께 보입니다.
- **스캐너 + 카메라 선택**: HID 바코드 스캐너(USB 스캐너)와 휴대폰/노트북 카메라 중 선택해서 QR을 읽을 수 있습니다. 잘못된 QR을 찍으면 화면에 경고가 뜹니다.
- **수동 입력**: QR이 안 읽힐 때 참가자 ID를 직접 입력해 체크인할 수 있습니다.
- **식사 체크인(Meal Check-in)**: 식사 시간대별로 따로 체크인할 수 있습니다.
- **식사 시간 설정**: **사이드바 → Settings → Meal Times** 에서 식사 시간대를 직접 설정합니다.
- **통계(Stats) 페이지**: 전체 체크인 현황을 한 번에 볼 수 있습니다.

**이렇게 쓰세요**
1. 행사장에서 체크인을 시작할 때 먼저 **세션을 시작**하세요.
2. 스캐너 또는 카메라로 참가자 QR(E-Pass)을 스캔합니다.
3. 식사 체크인은 별도 메뉴에서 진행합니다. (식사 시간은 미리 Settings에서 설정)
4. 다 끝나면 **세션을 종료**하세요. 세션 목록에서 나중에 다시 확인할 수 있습니다.

> 💡 모든 체크인 화면에 **뒤로 가기 버튼**이 생겨 이동이 편해졌습니다.

## 2. 💳 새 결제 수단 — 현장 결제

등록자가 선택할 수 있는 결제 방법에 **현장 결제(현장에서 결제)** 가 추가되었습니다. (기존 Zelle와 같은 "수동 결제" 그룹)

**동작 방식**
- 등록자가 현장 결제를 선택하면 등록이 **제출됨(SUBMITTED)** 상태가 되고, 수동 결제 할인이 적용됩니다.
- 이때는 아직 **결제 대기(PENDING)** 상태이며, E-Pass는 비활성 상태로 발급됩니다.
- **참가자가 행사장에 도착해 현장에서 결제하면, 관리자가 결제를 확인 처리**합니다. 그 시점에 E-Pass가 활성화됩니다.

**이렇게 쓰세요**
- 현장 결제로 등록한 사람은 도착 시 결제를 받고, 관리자 화면에서 **결제 확인(승인)** 처리하세요. 처리하면 자동으로 E-Pass가 활성화되고 확인 이메일이 나갑니다.

> ⚙️ 참고: 결제 수단은 코드 설정으로 켜고 끕니다. 현재 **현장 결제는 켜짐**, **수표(Check)는 꺼짐** 상태입니다.

## 3. 🔗 카드 결제 링크 (미결제 등록자용)

아직 결제하지 않은(SUBMITTED) 등록자에게 **로그인 없이 카드로 결제할 수 있는 링크**를 보낼 수 있습니다.

**이렇게 쓰세요**
1. **등록 상세 화면**을 엽니다. (등록 상태가 **SUBMITTED**일 때만 버튼이 보입니다.)
2. **"카드 결제 링크 생성"** 버튼을 누릅니다.
3. 생성된 링크를 복사해 등록자에게 전달합니다. (문자/이메일 등)
4. 등록자는 링크를 열어 로그인 없이 카드로 결제합니다.

**중요한 점**
- 이 링크로 결제하면 **카드 결제이므로 정가(수동 결제 할인 미적용)** 가 청구됩니다. (현장/Zelle 할인 가격이 아니라 원래 카드 가격)
- 결제가 완료되면 자동으로 **PAID** 상태로 바뀌고, E-Pass가 활성화되며, 확인 이메일이 발송됩니다.
- 링크는 **일회용**입니다. 결제가 끝나면 사용할 수 없게 됩니다.

## 4. 🔄 참가자 그룹 이전(Transfer) 방식 변경

참가자를 다른 등록 그룹으로 옮기는 방식이 더 안전하게 바뀌었습니다.

**무엇이 달라졌나**
- 이전(Transfer)하면 참가자가 대상 그룹에 **새로 복제(새 참가자 코드 발급)** 됩니다.
- **원래 등록의 결제 기록은 그대로 보존**됩니다. (정산·환불 시 꼬이지 않도록)
- 이제 **그룹의 마지막 한 명도 이전**할 수 있습니다.

**화면에서 보이는 것**
- 원래 등록에는 **"Transferred Out"** 섹션이 표시됩니다.
- 옮겨 온 참가자에는 **"from {원래 코드}"** 배지가 붙어, 어디서 왔는지 알 수 있습니다.

## 5. 💵 환불 처리 정확도 개선 (중요)

부분 환불(일부만 환불) 계산이 정확해졌습니다.

- **부분 환불 시 Stripe 처리 수수료가 비율대로 자동 차감**됩니다. (Stripe는 부분 환불 시 수수료를 돌려주지 않기 때문에, 그동안 교회가 떠안던 손해를 막습니다.)
- 누적 환불 한도 계산이 "고객이 실제로 받은 금액" 기준으로 바뀌어, 정당한 환불이 막히던 문제가 해결되었습니다.
- **Zelle/현장 결제 같은 수동 결제의 환불**은 라벨이 "수동 환불"로 표시됩니다. (Stripe 자동 환불이 아니라 직접 돌려줘야 한다는 의미)

> 💡 등록 상세의 **"New Total"** 은 관리자가 의도한 환불(총액 기준)을 반영합니다. 고객이 실제 받은 금액은 환불 기록에서 확인하세요.

## 6. 🆓 무료($0) 등록 자동 처리

총액이 $0인 무료 등록이 이제 정상적으로 처리됩니다.

- 예전에는 무료 등록이 **DRAFT(작성 중)** 에 머물러 **관리자 목록에 안 보이고, E-Pass·확인 이메일도 없던** 문제가 있었습니다.
- 이제 무료 등록도 자동으로 **제출됨(SUBMITTED)** 상태가 되어 목록에 보이고, 승인(APPROVED) 처리할 수 있습니다.
- 무료 등록은 Zelle/현장 결제처럼 **관리자 승인 전까지 대기 상태**로 표시됩니다.

## 7. 🔍 감사 로그(Audit Logs) 검색 & 정렬 개선

**감사 로그** (사이드바 → Audit Logs)
- **검색창** 추가: 작업 종류, 대상, **사용자 이메일**, **등록 확인 코드**로 검색할 수 있습니다.
- **페이지네이션**(25개씩) + 전체 개수 표시.
- 작업 종류·대상 유형 **드롭다운 필터** 추가.

**등록 목록(Registrations) 정렬**
- **"No."(등록번호)** 컬럼이 이제 **숫자 순서**로 정렬됩니다. (예전엔 "10"이 "2"보다 앞에 오던 문제 해결)

## 8. 📝 "타인 대신 등록" 화면 변경

**사이드바 메뉴 이름이 "Manual Registration" → "Register for Others"** 로 바뀌었습니다.

- 이제 관리자가 대신 등록할 때도 **일반 참가자와 똑같은 등록 마법사**를 사용합니다. (별도의 옛 관리자 전용 양식은 제거)
- **부서 → 등록 그룹 자동 전환**: 대표자의 부서를 바꾸면, 그 부서에 연결된 등록 그룹으로 자동 전환됩니다. (예: 한사모 대표 → 자동으로 올바른 그룹, "General" 아님) 자동 변경되면 안내 메시지가 표시됩니다.

## 9. 🚐 셔틀 기사(Shuttle Driver) 전용 계정

공항 셔틀 기사 권한만 가진 계정은 **공항 메뉴만** 보이도록 제한됩니다.

- 사이드바에 **"Airport"** 메뉴 하나만 표시됩니다.
- 다른 관리자 페이지에 접근하면 자동으로 공항 화면으로 돌아갑니다.
- 로고를 눌러도 공항 화면으로 이동합니다.

> 💡 셔틀 기사에게 계정을 줄 때 활용하세요. 다른 민감한 정보는 보이지 않습니다.

## 10. ✉️ 이메일 "E-Pass 보기" 링크 개선

확인/E-Pass 이메일의 **"View E-Pass(E-Pass 보기)"** 버튼이 개선되었습니다.

- 이제 **항상 공개 페이지(/epass)로 연결**됩니다. → 회원이 아닌 사람도 열 수 있습니다. (예전엔 회원 전용 페이지로 연결돼 비회원이 못 여는 경우가 있었음)
- 토큰이 없거나 비활성인 참가자도 메일 발송 시 자동으로 토큰이 만들어져, **모든 참가자가 E-Pass를 받을 수 있습니다.**
- 버튼이 커져서 **휴대폰에서 누르기 쉬워졌습니다.**
